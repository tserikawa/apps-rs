## ToDoリストの仕様

### 基本機能
- タスクの追加
- タスクの完了/未完了の切り替え
- タスクの削除

### UI構成
- テキスト入力欄 ＋ `追加`ボタン（上部）
- タスク一覧（チェックボックス ＋ タスク名 ＋ 削除ボタン）
- フィルターボタン：`すべて` `未完了` `完了済み`

### 動作仕様
- テキスト入力後に`追加`ボタンまたはEnterキーでタスクを追加
- 空文字は追加できない
- チェックボックスで完了/未完了をトグル
- 完了済みタスクはテキストに取り消し線を表示
- フィルターで表示するタスクを切り替える

### あえて含めないもの
- タスクの編集
- 並び替え（ドラッグ&ドロップ）
- データの永続化（アプリを閉じると消える）
- 優先度・期限などの属性

---

計算機と比べてRust的に新しく登場する概念は以下になります。

- `struct`でタスクを表現（計算機より自然な形で登場）
- `enum`でフィルター状態を管理
- `Vec<T>`への追加・削除・イテレータ操作
